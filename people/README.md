# People

One page per person who **recurs across more than one project or thread**. This is the connective tissue the project files can't hold — the hub that links a person to every place they show up.

## Who earns a page

The test: *do I re-derive who-this-is or what-they-touch more than once?* If yes, they get a page. If someone appears in exactly one project and nowhere else, leave them in that project's file — don't promote them here.

## What a page holds

Identity, relationship to Rick / Revolution Group, and **links out** to the project files and decisions where they appear. Keep it thin — the page is an index to the threads, not a biography. Facts that belong to a project (deal terms, interview scores) stay in the project file; this page links to them.

See [_template.md](_template.md) for the shape.

## Scope guardrail

Work / Revolution Group / TSD / EO contacts only. No personal-life or medical/health context — that's never ingested into this system (see [../inbox/README.md](../inbox/README.md)).
